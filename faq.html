FAQ / Troubleshooting

Preguntas frecuentes y solución de problemas comunes con ZER0.

Preguntas frecuentes

¿ZER0 funciona en distribuciones que no son Arch?

ZER0 está diseñado y probado específicamente para Arch Linux y sus derivadas (Manjaro, EndeavourOS, Garuda, etc.). Algunas funciones, especialmente los alias de pacman, no funcionarán en distribuciones basadas en Debian/Fedora/etc.

Puedes usarlo en otras distros bajo tu propio riesgo — el REPL base y los alias de git/python deberían funcionar, pero los alias específicos de sistema no.

¿Puedo usar ZER0 con zsh en vez de bash?

Sí. El instalador detecta automáticamente si usas bash o zsh y agrega los alias al archivo correcto (~/.bashrc o ~/.zshrc).

Si usas otra shell (fish, etc.), necesitas agregar manualmente el alias en tu archivo de configuración de shell:

alias zero='/usr/local/bin/zer0'
alias -Z='zero'
¿Mis alias personalizados se borran al actualizar ZER0?

No. Los alias personalizados se guardan en ~/.config/zer0/config.toml o ~/.config/zer0/aliases.toml, que son archivos de usuario y no se tocan durante las actualizaciones.

¿Cómo actualizo ZER0 a la última versión?

Desde el REPL:

zero --update

O vuelve a ejecutar el instalador — detecta que ya está instalado y solo actualiza los archivos del sistema:

bash <(curl -fsSL https://raw.githubusercontent.com/LogLabsHQ/ZER0/main/install.sh)

Problemas comunes

El comando zero no se reconoce después de instalar

El instalador añade los alias a tu ~/.bashrc o ~/.zshrc, pero necesitas recargar la sesión para que surtan efecto. Prueba:

source ~/.bashrc   # si usas bash
source ~/.zshrc    # si usas zsh

O simplemente abre una nueva terminal. Si el problema persiste, verifica que las líneas de alias estén al final de tu archivo de configuración de shell.

El REPL muestra caracteres extraños / rompe los colores

Esto ocurre cuando la terminal no soporta ANSI o Unicode correctamente. Desactiva los colores en la config:

[zer0]
colors = false

Asegúrate también de que tu terminal tenga una fuente con soporte para caracteres Unicode (recomendamos JetBrains Mono, Nerd Fonts o similar).

python no se encuentra al usar alias de Python

En Arch Linux, Python 3 se llama python (no python3). Si ves este error, asegúrate de tener Python instalado:

sudo pacman -S python

Si usas el Python embebido de ZER0 y encuentras problemas, reporta el issue en GitHub.

La instalación falla con error de permisos

El instalador puede necesitar escribir en /usr/local/bin. Si falla por permisos, prueba ejecutándolo explícitamente con los permisos correctos. El script solicitará sudo automáticamente cuando lo necesite — no corras sudo bash install.sh directamente.

Si el problema persiste, abre un issue en GitHub con el output completo del error.

FAQ / Troubleshooting

Frequently asked questions and solutions to common ZER0 issues.

Frequently Asked Questions

Does ZER0 work on non-Arch distributions?

ZER0 is designed and tested specifically for Arch Linux and its derivatives (Manjaro, EndeavourOS, Garuda, etc.). Some features, especially the pacman aliases, won't work on Debian/Fedora-based distros.

You can use it on other distros at your own risk — the base REPL and git/python aliases should work, but system-specific aliases won't.

Can I use ZER0 with zsh instead of bash?

Yes. The installer automatically detects whether you use bash or zsh and adds the aliases to the correct file (~/.bashrc or ~/.zshrc).

For other shells (fish, etc.), you need to manually add the alias to your shell config file:

alias zero='/usr/local/bin/zer0'
alias -Z='zero'
Are my custom aliases deleted when I update ZER0?

No. Custom aliases are saved in ~/.config/zer0/config.toml or ~/.config/zer0/aliases.toml, which are user files and are not touched during updates.

How do I update ZER0 to the latest version?

From the REPL:

zero --update

Or re-run the installer — it detects that ZER0 is already installed and only updates system files:

bash <(curl -fsSL https://raw.githubusercontent.com/LogLabsHQ/ZER0/main/install.sh)

Common Issues

The zero command isn't recognized after installation

The installer adds aliases to your ~/.bashrc or ~/.zshrc, but you need to reload the session for them to take effect. Try:

source ~/.bashrc   # if using bash
source ~/.zshrc    # if using zsh

Or simply open a new terminal. If the issue persists, verify the alias lines are at the bottom of your shell config file.

The REPL shows garbled characters / broken colors

This happens when the terminal doesn't properly support ANSI or Unicode. Disable colors in the config:

[zer0]
colors = false

Also make sure your terminal uses a font with Unicode support (we recommend JetBrains Mono, Nerd Fonts, or similar).

python not found when using Python aliases

On Arch Linux, Python 3 is called python (not python3). If you see this error, make sure Python is installed:

sudo pacman -S python

If you're using ZER0's embedded Python and encounter issues, please report them on GitHub.

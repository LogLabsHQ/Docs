  • Open source — MIT License. Sin dependencias ocultas.

ZER0

v1.0.3MIT LicenseArch Linux

Command alias manager for Arch Linux with a built-in interactive REPL. Streamline your workflow with 80+ default shortcuts, fully customizable and bilingual (ES/EN).

What is ZER0?

ZER0 acts as a productivity layer on top of your terminal. Instead of memorizing or typing long commands, you launch the ZER0 REPL and use short, semantic shortcuts to operate your Arch system.

Key Features

  • Interactive REPL — Persistent session with history, autocomplete, and colors.
  • 80+ default shortcuts — Covers pacman, git, systemd, networking, and more.
  • Bilingual ES/EN — Messages and help in Spanish and English.
  • Simple installation — Single install.sh file with embedded Python.
  • Custom aliases — Add, edit, or remove shortcuts from the REPL or config file.
  • Open source — MIT License. No hidden dependencies.

How to Open ZER0

zero
-Z
$Z

All three are exactly equivalent: they open the ZER0 interactive REPL.

Instalación

ZER0 se instala con un solo comando. El instalador descarga y configura todo automáticamente.

Requisitos

  • Arch Linux (o derivadas: Manjaro, EndeavourOS, etc.)
  • bash — viene por defecto en Arch
  • curl — para descargar el instalador
  • Conexión a Internet
Nota
ZER0 está diseñado específicamente para Arch Linux. No se garantiza soporte oficial en otras distribuciones.

Instalación recomendada

bash <(curl -fsSL https://raw.githubusercontent.com/LogLabsHQ/ZER0/main/install.sh)

El script verificará los requisitos, descargará ZER0, configurará los alias (zero, -Z, $Z) y creará el archivo de config en ~/.config/zer0/.

Instalación manual

# 1. Clonar el repositorio
git clone https://github.com/LogLabsHQ/ZER0.git
cd ZER0

# 2. Ejecutar el instalador
bash install.sh

Verificar la instalación

zero --version

Deberías ver: ZER0 v1.0.3 — LogLabs

Tip
Si el comando zero no se reconoce tras instalar, ejecuta source ~/.bashrc (o ~/.zshrc) o abre una nueva terminal.

Desinstalación

zero --uninstall

Configuración

Personaliza ZER0: agrega alias propios, cambia idioma, ajusta el editor y más.

Archivo de configuración

ZER0 guarda su configuración en:

~/.config/zer0/config.toml

Ábrelo desde el REPL con edit, o directamente:

nvim ~/.config/zer0/config.toml

Opciones globales

[zer0]
lang         = "es"      # Idioma: "es" o "en"
editor       = "nano"    # Editor para el comando `edit`
prompt       = "zer0 >"  # Texto del prompt
history      = true      # Guardar historial
history_size = 500       # Máximo de entradas
colors       = true      # Colores ANSI
OpciónTipoDefaultDescripción
langstring"es"Idioma del REPL.
editorstring"nano"Editor del comando edit.
promptstring"zer0 >"Texto del prompt.
historybooltrueActivar historial.
history_sizeint500Máximo de entradas.
colorsbooltrueColores ANSI.

Alias personalizados

[aliases]
myupdate = "sudo pacman -Syu && yay -Syu"
proj     = "cd ~/projects"
serve    = "python -m http.server 8080"

Agregar alias desde el REPL

add myalias "comando que quieres ejecutar"
Atención
Los alias personalizados no pueden sobreescribir los alias del sistema de ZER0 (help, exit, list, etc.).

Recargar la configuración

reload

FAQ / Troubleshooting

Preguntas frecuentes y solución de problemas comunes con ZER0.

Preguntas frecuentes

¿ZER0 funciona en distribuciones que no son Arch?

ZER0 está diseñado y probado para Arch Linux y sus derivadas. Los alias de pacman no funcionarán en otras distros, aunque el REPL base y los alias de git/python sí deberían hacerlo.

¿Puedo usar ZER0 con zsh?

Sí. El instalador detecta automáticamente si usas bash o zsh y configura el archivo correcto (~/.bashrc o ~/.zshrc).

¿Mis alias personalizados se borran al actualizar?

No. Los alias personalizados se guardan en ~/.config/zer0/, que es un directorio de usuario y no se toca durante las actualizaciones.

¿Cómo actualizo ZER0?
zero --update

O vuelve a ejecutar el instalador — detecta que ya está instalado y solo actualiza los archivos del sistema.

Problemas comunes

El comando zero no se reconoce tras instalar

Recarga tu shell o abre una nueva terminal:

source ~/.bashrc   # bash
source ~/.zshrc    # zsh
El REPL muestra caracteres extraños / colores rotos

Desactiva los colores en la config:

[zer0]
colors = false

Asegúrate de usar una fuente con soporte Unicode (JetBrains Mono, Nerd Fonts, etc.).

La instalación falla con error de permisos

No corras el instalador con sudo directamente — el script pedirá sudo automáticamente cuando lo necesite. Si el problema persiste, abre un issue en GitHub.

FAQ / Troubleshooting

Frequently asked questions and solutions to common ZER0 issues.

Frequently Asked Questions

Does ZER0 work on non-Arch distributions?

ZER0 is designed for Arch Linux and derivatives. Pacman aliases won't work on other distros, but the base REPL and git/python aliases should.

Can I use ZER0 with zsh?

Yes. The installer automatically detects whether you use bash or zsh and configures the correct file (~/.bashrc or ~/.zshrc).

Are my custom aliases deleted when I update?

No. Custom aliases are saved in ~/.config/zer0/, a user directory not touched during updates.

How do I update ZER0?
zero --update

Or re-run the installer — it detects the existing installation and only updates system files.

Common Issues

The zero command isn't recognized after install

Reload your shell or open a new terminal:

source ~/.bashrc   # bash
source ~/.zshrc    # zsh
The REPL shows garbled characters / broken colors

Disable colors in config:

[zer0]
colors = false

Make sure your terminal uses a Unicode font (JetBrains Mono, Nerd Fonts, etc.).

Installation fails with a permissions error

Don't run the installer with sudo directly — the script will ask for sudo when needed. If the issue persists, open an issue on GitHub.

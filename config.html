Configuración

Personaliza ZER0 a tu gusto: agrega alias propios, cambia el idioma por defecto, ajusta el editor y más.

Archivo de configuración

ZER0 guarda su configuración en:

~/.config/zer0/config.toml

Puedes abrirlo directamente desde el REPL con:

edit

O con tu editor preferido:

nano ~/.config/zer0/config.toml
# o
nvim ~/.config/zer0/config.toml

Opciones globales

Ejemplo de configuración base:

[zer0]
lang     = "es"          # Idioma por defecto: "es" o "en"
editor   = "nano"        # Editor para el comando `edit`
prompt   = "zer0 >"      # Texto del prompt en el REPL
history  = true          # Guardar historial de comandos
history_size = 500       # Número de entradas en el historial
colors   = true          # Activar colores en el REPL
OpciónTipoPor defectoDescripción
langstring"es"Idioma del REPL (es o en).
editorstring"nano"Editor que abre el comando edit.
promptstring"zer0 >"Texto del prompt del REPL.
historybooltrueActivar/desactivar historial.
history_sizeint500Máximo de entradas en el historial.
colorsbooltrueColores ANSI en el REPL.

Alias personalizados

Agrega tus propios alias en la sección [aliases]:

[aliases]
# Formato: alias = "comando completo"
myupdate = "sudo pacman -Syu && yay -Syu"
proj     = "cd ~/projects"
serve    = "python -m http.server 8080"
ports80  = "sudo ss -tulpn | grep ':80'"

Agregar alias desde el REPL

También puedes agregar alias sin editar el archivo manualmente:

add myalias "comando que quieres ejecutar"

ZER0 lo guarda automáticamente en el archivo de config.

Atención
Los alias personalizados no pueden sobreescribir los alias del sistema de ZER0 (como help, exit, list, etc.). Si intentas hacerlo, ZER0 mostrará un error.

Aliases del archivo de aliases

Si prefieres separar tus alias en un archivo aparte:

~/.config/zer0/aliases.toml

ZER0 lo carga automáticamente si existe. El formato es el mismo que la sección [aliases] en config.toml.

Recargar la configuración

Después de editar el archivo, recarga la config desde el REPL sin reiniciar:

reload

Configuration

Customize ZER0 to your liking: add your own aliases, change the default language, adjust the editor, and more.

Config File

ZER0 stores its configuration at:

~/.config/zer0/config.toml

You can open it directly from the REPL with:

edit

Or with your preferred editor:

nano ~/.config/zer0/config.toml
# or
nvim ~/.config/zer0/config.toml

Global Options

Example base configuration:

[zer0]
lang     = "en"          # Default language: "es" or "en"
editor   = "nano"        # Editor for the `edit` command
prompt   = "zer0 >"      # REPL prompt text
history  = true          # Save command history
history_size = 500       # Number of history entries
colors   = true          # Enable colors in the REPL
OptionTypeDefaultDescription
langstring"es"REPL language (es or en).
editorstring"nano"Editor opened by the edit command.
promptstring"zer0 >"REPL prompt text.
historybooltrueEnable/disable history.
history_sizeint500Maximum history entries.
colorsbooltrueANSI colors in the REPL.

Custom Aliases

Add your own aliases in the [aliases] section:

[aliases]
# Format: alias = "full command"
myupdate = "sudo pacman -Syu && yay -Syu"
proj     = "cd ~/projects"
serve    = "python -m http.server 8080"
ports80  = "sudo ss -tulpn | grep ':80'"

Adding aliases from the REPL

You can also add aliases without manually editing the file:

add myalias "command you want to run"

ZER0 saves it automatically to the config file.

Warning
Custom aliases cannot overwrite ZER0 system aliases (like help, exit, list, etc.). If you try, ZER0 will show an error.

Separate Aliases File

If you prefer to keep your aliases in a separate file:

~/.config/zer0/aliases.toml

ZER0 loads it automatically if it exists. The format is the same as the [aliases] section in config.toml.

Reloading Config

After editing the file, reload the config from the REPL without restarting:

reload
